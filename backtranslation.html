Backtranslation for RL Environments

synthetic data
2025-09-17 • status: in-progress • importance: medium

Creating environments for RL on verifiable rewards (RLVR) does not require in-depth understanding of the data. At minimum, it requires taking existing "answers" and fitting them with appropriate questions. Consider that a model doesn't have to be able to get questions on coding problems correct to be able to take a textbook with given answers and simply format it into appropriate JSON for a separate training environment. In a 2023 paper from Meta, researchers discuss repurposing the backtranslation method for training translation models to create instruction-tuning data for language models. Taking the mass of data on the internet and converting answers to given questions on QA forums and similar, a model can simply convert that to the proper "prompt : answer" pair format required for instruction-tuning, and additionally come up with questions to befit answers.

In RLVR, you need two things: questions and a way to verify if an answer is correct to that question. You could scrape leetcode for problems and tests, but if you only had tests, you could come up with problems for those tests and if you only had problems you could come up with tests for those problems. These three approaches would at minimum supply data for correctly formatted RL environments. Accuracy and verifying whether tests verify problems is a separate problem to finding data on its own but necessary to address for an approach like this.

Before addressing that, it's worth expanding what can be considered potential training data because this formulation is a big expansion.
